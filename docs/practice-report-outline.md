# 实践报告大纲 — 基于多智能体的对话式社交匹配系统

## 一、项目概述

### 1.1 课题背景
- 传统找搭子平台的三大痛点：交互门槛高、画像维度有限、社交启动难
- 大模型驱动的多智能体协作框架的技术契机

### 1.2 项目目标
- 构建"零表单、全对话"的社交匹配系统
- 实现画像隐式采集 → 智能匹配 → 破冰辅助的全流程闭环

### 1.3 技术路线
- Python/FastAPI + OpenAI兼容API + ChromaDB + React

## 二、系统设计

### 2.1 总体架构
- 三层架构：API网关 → 服务编排 → Agent层 → 基础设施
- 多Agent协作模式与通信协议

### 2.2 核心Agent设计

#### 2.2.1 画像采集Agent
- 对话状态机（7阶段）
- Prompt工程：结构化偏好提取 + 性格分析
- 动态画像构建与版本管理

#### 2.2.2 匹配决策Agent
- 画像向量化编码（text-embedding-3-small, 1536维）
- 加权多维评分公式: total = 0.50×兴趣 + 0.30×性格 + 0.20×需求
- 硬约束规则引擎（支持热更新）
- MMR多样性重排算法

#### 2.2.3 撮合辅助Agent
- 三风格破冰话术（幽默/正式/温暖）
- 可解释推荐（自然语言匹配理由）
- 活动建议生成

### 2.3 数据模型设计
- 9张核心数据表
- JSON字段存储灵活画像结构
- 画像版本历史 ≥10版

### 2.4 API设计
- RESTful接口，4大模块，20+个端点

## 三、关键技术实现

### 3.1 LLM适配器模式
- 抽象层设计：支持OpenAI/Ollama/其他兼容API无缝切换
- 结构化输出：通过JSON Schema约束提取格式

### 3.2 向量化与相似度计算
- 画像→自然语言文本→Embedding的转换策略
- 余弦相似度 + 加权多维评分

### 3.3 规则引擎
- 热配置机制
- 硬约束过滤 + 软偏好加权

### 3.4 前端实现
- React + TypeScript + Tailwind CSS
- 即时通讯风格的对话界面
- 实时画像卡片 + 匹配结果可视化

## 四、系统测试

### 4.1 单元测试
- 评分算法、规则引擎、MMR算法的独立验证

### 4.2 集成测试
- 对话→匹配→破冰全链路测试
- Mock LLM降低测试成本

### 4.3 性能指标
- 对话端到端响应 P95 < 3s
- 匹配计算 < 500ms
- 向量检索 < 200ms

## 五、应用场景分析

### 5.1 企业内部社交
- 跨部门协作搭子、新员工融入匹配

### 5.2 兴趣社群运营
- 运动/学习/旅行等垂直社区的搭伴匹配

### 5.3 会展参会者匹配
- 行业会议的商务社交对接

## 六、总结与展望

### 6.1 项目成果
- 完整可运行的多Agent社交匹配系统
- 含前端界面、后端API、管理后台

### 6.2 创新点
- 零表单交互：用户全程通过自然对话完成画像构建
- 可解释推荐：透明的匹配理由 + 得分分解
- 多Agent协作：分工明确、可独立扩展

### 6.3 未来改进
- 引入反馈学习：根据匹配结果微调画像权重
- 图神经网络：基于社交网络结构的协同过滤
- 多模态画像：支持语音/图片输入
